PROBLEM: Users lose their accounts after 30 minutes in production

ROOT CAUSE: Using Render.com's free tier PostgreSQL database
            Free tier databases are EPHEMERAL (temporary) and get deleted

SOLUTION: Switch to a persistent database (15 minutes to fix)

================================================================================
                         QUICK FIX (Choose One)
================================================================================

OPTION 1: Supabase (FREE - Recommended)
─────────────────────────────────────────
1. Go to https://supabase.com → Sign up
2. Create new project
3. Settings → Database → Copy connection string
4. Render dashboard → Your backend → Environment
5. Add: DATABASE_URL = [paste connection string]
6. Save Changes (service redeploys)
7. Wait 2-3 minutes
8. DONE - Data now persists!

OPTION 2: Render Standard PostgreSQL ($15/month)
──────────────────────────────────────────────────
1. Render dashboard → New + → PostgreSQL
2. Name: langgamit-db
3. Plan: STANDARD (NOT free!)
4. Create Database
5. Wait 5-10 minutes
6. Copy Internal Database URL
7. Your backend → Environment
8. Update: DATABASE_URL = [paste URL]
9. Save Changes
10. Wait 2-3 minutes
11. DONE - Data now persists!
